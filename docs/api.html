API

Version 0.2.6

This page provides a detailed reference for the public API surface used in tutorials and scripts. For each method below you get: purpose, full parameter list, and direct source link.

Simulation

Module: ncpi/Simulation.py

  • run_script(script_path, param_path, output_folder): source
    Run a Python script with a parameter module and output folder.
    Parameters: script_path script filename, param_path parameter filename, output_folder destination directory.
  • Simulation.__init__(param_folder, python_folder, output_folder): source
    Create a simulation runner bound to folders for parameter files, executable scripts, and outputs.
    Parameters: param_folder params directory, python_folder scripts directory, output_folder output directory (created if missing).
  • Simulation.network(script_path, param_path): source
    Execute network-generation stage.
    Parameters: script_path script under python_folder, param_path params under param_folder.
  • Simulation.simulate(script_path, param_path): source
    Execute simulation stage with the same path semantics as network.
    Parameters: script_path simulation script filename relative to python_folder (or absolute path), param_path parameter filename relative to param_folder (or absolute path).
  • Simulation.analysis(script_path, param_path): source
    Execute analysis stage with the same path semantics as network.
  • Simulation.bundle_output_folder(output_folder, output_filename='simulation.pkl', remove_source_files=False): source
    Pack core simulation artifacts into one pickle payload for downstream use.
    Parameters: output_folder folder to bundle, output_filename bundle name, remove_source_files delete source pickle files after bundling.
  • Simulation.bundle_outputs(output_filename='simulation.pkl', remove_source_files=False): source
    Convenience wrapper around bundle_output_folder using the instance output folder.
    Parameters: output_filename name of the bundled pickle written inside output_folder, remove_source_files remove per-run pickle files after successful bundling.

FieldPotential

Module: ncpi/FieldPotential.py

  • FieldPotential.__init__(): source
    Initialize internal kernel/proxy registries and lazy dependency handles.
  • FieldPotential.roll_with_zeros(arr, shift): source
    Shift a 1D array and zero-fill exposed elements.
    Parameters: arr input array, shift positive/negative integer shift.
  • FieldPotential.create_kernel(MC_folder, params, biophys, dt, tstop, *, output_sim_path=None, electrodeParameters=None, CDM=True, probes=None, cdm_probe=None, mean_nu_X=None, Vrest=None, t_X=None, tau=None, g_eff=None, n_ext=None, weights=None): source
    Build multicompartment kernels for CDM/LFP workflows.
    Parameters: MC_folder model folder, params network parameter object/module, biophys biophysical setup list, dt/tstop simulation timing, output_sim_path optional path to simulation outputs for inferred rates/resting state, electrodeParameters LFP probe settings, CDM include CDM probe, probes direct probe override, cdm_probe CDM probe class/name, mean_nu_X/Vrest manual alternatives when no output path is provided, t_X/tau/g_eff/n_ext/weights kernel and synaptic overrides.
  • FieldPotential.compute_cdm_lfp_from_kernels(kernels, spike_times, dt, tstop, population_sizes=None, transient=0.0, probe='KernelApproxCurrentDipoleMoment', component=2, mode='same', scale=1.0, aggregate=None): source
    Convolve spike trains with precomputed kernels to obtain CDM/LFP traces.
    Parameters: kernels kernel dictionary, spike_times spikes by population, dt/tstop output grid, population_sizes optional normalization, transient discard pre-window, probe probe key, component dipole component selection, mode convolution mode, scale scalar multiplier, aggregate optional sum output.
  • FieldPotential.compute_MEEG(CDM, dipole_locations=None, sensor_locations=None, model=None, model_kwargs=None, align_to_surface=True): source
    Apply EEG/MEG forward models to CDM trajectories.
    Parameters: CDM dipole time series (single or multi-dipole), dipole_locations dipole positions, sensor_locations sensor positions (non-NYHead models), model forward model name, model_kwargs constructor args, align_to_surface NYHead orientation alignment.
  • FieldPotential.compute_proxy(method, sim_data, sim_step, *, excitatory_only=None): source
    Compute scalar proxy signals (FR/AMPA/GABA/Vm/I/I_abs/LRWS/ERWS1/ERWS2).
    Parameters: method proxy key, sim_data required simulation arrays map, sim_step timestep in ms (mandatory for delayed proxies), excitatory_only key-resolution preference for excitatory/all channels.

Features

Module: ncpi/Features.py

  • Features.__init__(method='catch22', params=None): source
    Configure feature extraction backend and options.
    Parameters: method one of catch22, specparam, dfa, fEI, hctsa; params method-specific options (includes normalize).
  • Features.catch22(sample): source
    Compute the 22 canonical catch22 features from a 1D sample.
    Parameters: sample input time-series.
  • Features.specparam(sample=None, *, freqs=None, power_spectrum=None, fs=None, freq_range=None, welch_kwargs=None, model_kwargs=None, select_peak=None, debug=None, metric_thresholds=None, metric_policy=None): source
    Fit an aperiodic/periodic spectral model from time-domain or PSD inputs.
    Parameters: sample time-domain signal (mutually exclusive with freqs/power_spectrum), freqs frequency axis, power_spectrum PSD values, fs sampling frequency (when sample used), freq_range fit range, welch_kwargs PSD options, model_kwargs spectral-model options, select_peak peak selection strategy, debug enable debug output, metric_thresholds fit-quality thresholds, metric_policy threshold handling (reject/flag).
  • Features.dfa(sample, *, sampling_frequency=None, fit_interval=None, compute_interval=None, overlap=None, frequency_range=None, spectrum_range=None, bad_idxes=None, input_is_envelope=None, filter_kwargs=None, trim_seconds=None, hilbert_n_fft=None): source
    Compute detrended fluctuation analysis features.
    Parameters: sample 1D/2D input, sampling_frequency Hz, fit_interval/compute_interval DFA windows, overlap overlapping windows flag, frequency_range band mode, spectrum_range spectrum mode, bad_idxes channel exclusions, input_is_envelope envelope flag, filter_kwargs MNE filter settings, trim_seconds edge trim, hilbert_n_fft Hilbert FFT length.
  • Features.fEI(sample, *, sampling_frequency=None, window_size_sec=None, window_overlap=None, dfa_value=None, dfa_threshold=None, dfa_fit_interval=None, dfa_compute_interval=None, dfa_overlap=None, frequency_range=None, spectrum_range=None, bad_idxes=None, input_is_envelope=None, filter_kwargs=None, trim_seconds=None, hilbert_n_fft=None): source
    Compute functional excitation-inhibition index from envelopes/spectra.
    Parameters: sample 1D/2D input, sampling_frequency Hz, window_size_sec/window_overlap fEI window config, dfa_value optional precomputed DFA, dfa_threshold minimum DFA threshold, dfa_fit_interval/dfa_compute_interval/dfa_overlap internal DFA settings, frequency_range/spectrum_range band mode, bad_idxes, input_is_envelope, filter_kwargs, trim_seconds, hilbert_n_fft.
  • Features.hctsa(samples, hctsa_folder, workers=None, return_meta=False): source
    Run MATLAB-based hctsa feature extraction.
    Parameters: samples collection of 1D series, hctsa_folder HCTSA installation path, workers MATLAB worker count, return_meta include metadata in output.
  • Features.compute_features(samples, n_jobs=None, chunksize=None, start_method='spawn', progress_callback=None, log_callback=None): source
    Parallel feature extraction entry point.
    Parameters: samples list/array of 1D signals, n_jobs worker count, chunksize dispatch chunk size, start_method multiprocessing method, progress_callback progress hook, log_callback log hook.

Inference

Module: ncpi/Inference.py

  • Inference.__init__(model, hyperparams=None): source
    Initialize sklearn or SBI backend based on model.
    Parameters: model regressor name or SBI key (NPE/NLE/NRE), hyperparams backend-specific configuration.
  • Inference._coerce_sample_shape(sample_shape): source
    Normalize scalar/tuple sample shape into torch-compatible shape object.
  • Inference._sample_posterior(posterior_obj, *, x=None, **sbi_eval_sampling_kwargs): source
    Draw posterior samples from SBI posterior objects for single or batched observations.
    Parameters: posterior_obj trained posterior object, x conditioning observation(s), sbi_eval_sampling_kwargs forwarded sampler kwargs.
  • Inference.add_simulation_data(features, parameters, *, append=False, copy=False): source
    Register training feature/target arrays, with optional append mode.
    Parameters: features feature matrix/vector, parameters target matrix/vector, append concatenate to existing data, copy force copying.
  • Inference.initialize_sbi(hyperparams): source
    Build an SBI inference object and estimator configuration.
    Parameters: hyperparams dict including required prior and optional inference_kwargs, estimator, estimator_kwargs, build_posterior_kwargs.
  • Inference.train(param_grid=None, *, n_splits=10, n_repeats=10, train_params=None, result_dir='data', scaler=None, seed=0, sklearn_verbose=None, sbi_eval_sampling_kwargs=None): source
    Train a model (single-fit or repeated CV search) and persist artifacts.
    Parameters: param_grid optional hyperparameter candidates, n_splits/n_repeats repeated CV settings, train_params extra SBI train args, result_dir artifact output folder, scaler transformer or bool flag, seed reproducibility seed, sklearn_verbose default estimator verbosity, sbi_eval_sampling_kwargs posterior-sampling kwargs used for SBI CV scoring.
  • Inference.predict(features, *, result_dir='data', scaler=False, n_jobs=None, chunksize=None, start_method='spawn', sbi_eval_sampling_kwargs=None): source
    Load trained artifacts and generate predictions from new feature rows.
    Parameters: features input feature rows, result_dir artifact location, scaler whether to apply saved scaler, n_jobs/chunksize/start_method sklearn parallelism settings, sbi_eval_sampling_kwargs posterior sampling controls.

Analysis

Module: ncpi/Analysis.py

  • extract_variables(formula): source
    Tokenize an R-style formula and extract variable-like terms.
    Parameters: formula expression like Y ~ group * epoch + (1|id).
  • Analysis.__init__(data): source
    Store an analysis data object (typically a pandas DataFrame for statistical methods).
    Parameters: data source container.
  • Analysis.lmer_tests(models, group_col=None, control_group=None, numeric=None, specs=None, *, posthoc=None, print_info=True, lrt=False, lrt_drop=None, return_model_info=False): source
    Fit lm/lmer models through R, run post-hoc tests, optionally run LRT and return model metadata.
    Parameters: models formula(s), group_col group field, control_group control level, numeric numeric variables, specs post-hoc specs, posthoc post-hoc policy/config, print_info print diagnostics, lrt run likelihood-ratio test, lrt_drop reduced-model terms, return_model_info include selection metadata.
  • Analysis.lmer_selection(full_model, numeric=None, *, crit=None, random_crit='BIC', fixed_crit='LRT', include=None, print_info=True): source
    Backward model selection using R buildmer for random and/or fixed terms.
    Parameters: full_model starting formula, numeric numeric variable list, crit single criterion override, random_crit random-effects criterion, fixed_crit fixed-effects criterion, include forced terms, print_info print selected model.
  • Analysis.cohend(*, control_group='HC', data_col='Y', data_index=-1, group_col='group', sensor_col='sensor', min_n=3, drop_zeros=True): source
    Compute sensor-wise Cohen’s d for each non-control group vs control.
    Parameters: control_group baseline level, data_col measurement column, data_index element index for sequence-valued rows, group_col group column, sensor_col sensor column, min_n minimum per-group samples, drop_zeros remove zero-valued rows.
  • Analysis.eeg_topomap(values=None, ch_names=None, *, position_offset, sensor_col='sensor', data_col='data', data_index=-1, montage='standard_1005', ch_type='eeg', sphere=0.28, cmap='jet', vmin=None, vmax=None, colorbar=True, colorbar_fmt='%0.2f', colorbar_label=None, colorbar_label_fontsize=None, colorbar_tick_fontsize=8, sensors=None, outlines=None, contours=None, res=300, extrapolate=None, image_interp=None, axes=None, show=True, title=None, names=None): source
    Render EEG scalp topography from passed values or from self.data.
    Parameters: values/ch_names explicit map input (or infer from data columns), position_offset mandatory (y,z) offset, sensor_col/data_col/data_index DataFrame mapping, montage/ch_type channel geometry, sphere/cmap/vmin/vmax style and limits, colorbar* colorbar controls, sensors/outlines/contours/res/extrapolate/image_interp MNE topomap options, axes/show/title/names display controls.
  • Analysis.meg_surface(values, *, atlas='dk', subject='fsaverage', subjects_dir=None, surface='inflated', hemisphere='both', views='lat', cmap='coolwarm', vmin=None, vmax=None, center=None, alpha=1.0, smoothing_steps=5, colorbar=True, colorbar_kwargs=None, cortex='low_contrast', background='white', size=(1000, 600), show=True, axes=None, auto_fetch=True, hcp_accept=False, verbose=None, **brain_kwargs): source
    Plot parcel-wise MEG values on cortical surfaces using MNE/FreeSurfer parcellations.
    Parameters: values one value per parcel, atlas parcellation name, subject/subjects_dir source anatomy, surface/hemisphere/views viewing geometry, cmap/vmin/vmax/center/alpha/smoothing_steps rendering style, colorbar/colorbar_kwargs color scale, cortex/background/size scene styling, show/axes matplotlib target behavior, auto_fetch auto-download fsaverage, hcp_accept HCP-MMP license acceptance, verbose MNE verbosity, brain_kwargs additional Brain constructor args.

EphysDatasetParser

Module: ncpi/EphysDatasetParser.py

  • CanonicalFields: source
    Declarative locator map for canonical parsed fields.
    Fields (parameters): data, fs, ch_names, time, data_domain, freqs, spectral_kind, epoch, metadata, table_layout, channel_columns, long_channel_col, long_value_col, long_time_col, array_axes.
  • ParseConfig: source
    Parser behavior configuration for loading, segmentation, epoching, normalization, and aggregation.
    Fields (parameters): fields, epoch_length_s, epoch_step_s, segment_t0_s, segment_t1_s, preload, pick_types, max_seconds, drop_bads, zscore, zscore_after_epoch, exclude_last_epoch, align_epoch_count_to_minimum, aggregate_over, aggregate_method, aggregate_labels, warn_unimplemented.
  • EphysDatasetParser.__init__(config=None): source
    Construct a parser instance with explicit configuration.
    Parameters: config optional ParseConfig (defaults applied when omitted).
  • EphysDatasetParser.parse(source): source
    Parse in-memory objects or file paths into canonical DataFrame rows.
    Parameters: source path or object (supports MNE objects, arrays, dict-like, DataFrame, and several file formats).
